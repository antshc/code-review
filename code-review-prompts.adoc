:toc:
:toclevels: 2

This document captures AI code review prompts exactly as reusable, copy-pasteable instruction blocks.
The wording is intentionally kept close to the original text.

== Core Code Review Prompts

=== General Review
[source,text]
----
Review the following code for readability, potential edge cases, and best practices.
----

[source,text]
----
Review the following code for potential bugs, performance bottlenecks, security vulnerabilities, and adherence to [Language/Framework] best practices.
Provide actionable suggestions and explain your reasoning.
----

=== Bug & Error Focus
[source,text]
----
This code produces [error/unexpected output].
Explain the root cause and provide a corrected version with improved error handling.
----

=== Performance & Optimization
[source,text]
----
Optimize this function for O(n) complexity.
----

[source,text]
----
Optimize this algorithm for time complexity.
Suggest improvements like caching or better data structures, and explain the trade-offs.
----

=== Readability & Style
[source,text]
----
Refactor this code to be more modular, readable, and follow the [Team Style Guide/PEP 8].
Improve variable names and break down complex logic.
----

=== Security
[source,text]
----
Analyze this code for common security flaws (e.g., SQL injection, XSS) and suggest specific mitigations.
----

== Advanced Techniques & Specific Prompts

=== Explain Legacy or Complex Code
[source,text]
----
Explain this legacy code to a junior dev.
----

[source,text]
----
Explain this [React/Python] code to me as a junior developer, detailing each line and its purpose,
then show an advanced implementation.
----

=== Test Generation
[source,text]
----
Generate unit tests using [framework].
----

[source,text]
----
Generate comprehensive unit tests for this function using Jest/Pytest,
covering edge cases and common inputs.
----

=== Documentation
[source,text]
----
Generate clean, Markdown-formatted API documentation from this code block,
explaining parameters, return types, and usage.
----

=== Step-by-Step Refactoring
[source,text]
----
Help me refactor this file.
Go step-by-step:
Step 1: Analyze structure.
Step 2: Suggest function extraction.
(Wait for "next" after each step.)
----

== Prompt Usage Tips (As Instructions)

[source,text]
----
Be specific: mention language, framework, and exact goals (e.g., "Python", "Django", "O(n) complexity").
----

[source,text]
----
Provide examples: include problematic input/output pairs or specific error messages.
----

[source,text]
----
Ask for trade-offs: request balanced advice on performance vs. readability.
----

[source,text]
----
Reference guides: attach your team's style guide for consistency.
----
